# NoClone (Pro Edition) 🚀

## Описание
**NoClone** — это мощный инструмент на Python с графическим интерфейсом (GUI) для молниеносного удаления дубликатов из огромных текстовых списков. Программа сравнивает входящие данные с локальной базой данных SQLite, обеспечивая мгновенную проверку на уникальность даже при наличии миллионов существующих записей.

Идеально подходит для специалистов по данным, арбитражников и разработчиков, работающих с большими списками ID, почт или ссылок, где требуется сохранять историю уникальных элементов в течение долгого времени.

---

## Ключевые особенности
- **Интерфейс на PySide6 (Qt6)**: Современный, интуитивно понятный дизайн.
- **Два режима работы**: 
  - **Ручной**: Просто вставьте текст в окно программы.
  - **Файловый**: Пакетная обработка файлов любого размера.
- **Оптимизация SQLite (WAL mode)**: Высокая скорость записи и чтения благодаря пакетной обработке (Batch Processing) и транзакциям.
- **Гибкие правила**: Автоматический перевод в нижний регистр, очистка от спецсимволов и фильтрация по длине.
- **Drag & Drop**: Просто перетащите `.txt` файл или базу `.db` в окно программы для мгновенной загрузки.
- **Оптимизация БД**: Встроенная функция `VACUUM` для сжатия базы данных и освобождения места на диске.

## Установка и запуск

### 1. Требования
- **Python 3.10** или выше.

### 2. С нуля (Windows)
1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/YourUsername/NoClone.git
   cd NoClone
   ```
2. **Создайте виртуальное окружение**:
   ```bash
   python -m venv .venv
   ```
3. **Активируйте его**:
   ```bash
   .venv\Scripts\activate
   ```
4. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```
5. **Запустите программу**:
   ```bash
   python main.py
   ```

### 3. Быстрый запуск для Windows
Просто используйте файл `NoClone_GUI.bat` для запуска без консоли (требует наличия настроенного окружения в папке `.venv`).

## Как пользоваться

### Ручной режим (Paste & Go)
1. Перейдите на вкладку **"Ручной режим"**.
2. Вставьте ваши строки в левое окно.
3. Нажмите **"Очистить дубликаты и добавить в базу"**.
4. Уникальные строки появятся справа, их можно скопировать одной кнопкой.

### Файловый режим
1. Перейдите на вкладку **"Файловый режим"**.
2. Укажите входной файл (или перетащите его в окно).
3. (Опционально) Укажите путь к базе `.db` и файлу результата.
4. Нажмите **"Начать обработку"**. По завершении `input.txt` будет очищен, а новые строки сохранены в отдельный файл.

### Правила обработки
В верхней части окна можно настроить:
- **К нижнему регистру**: Полезно для email или доменов.
- **Удалить спецсимволы**: Оставляет только буквы и цифры.
- **Мин. длина строки**: Пропускает слишком короткие записи.

## Архитектура и Структура
- **`main.py`**: Ядро программы и весь UI-код.
- **`IdsBD.db`**: База данных SQLite. Хранит только уникальные значения.
- **`logs/`**: Ежедневные логи всех операций.
- **`requirements.txt`**: Список необходимых библиотек (`PySide6`).

---

## Технические детали (для разработчиков)
- **Batch Processing**: Данные обрабатываются группами по 5000 записей, что минимизирует количество обращений к диску.
- **SQL Indexing**: Поле `content` является первичным ключом (`PRIMARY KEY`), что на уровне движка БД исключает дубликаты и ускоряет поиск.
- **Multithreading**: Вся обработка вынесена в отдельный поток (`QThread`), поэтому интерфейс не "зависает" даже при работе с файлами весом в несколько гигабайт.
